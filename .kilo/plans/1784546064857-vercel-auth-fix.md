# Plan: Fix Google + password auth on Vercel deploy

## Problem
After deploying to Vercel, both password (login/register) and Google OAuth authentication fail.
Root cause is the backend API is not actually being built/run by Vercel, plus env secrets
and OAuth callback config issues.

## Diagnosis (verified against code)
1. **API not executed on Vercel** — `vercel.json` has only `rewrites`, no `functions`/`builds`.
   `api/index.ts` is raw TypeScript importing `../src/server/app.ts`; Vercel Node runtime
   cannot run `.ts` and the built `dist/server.cjs` is not wired to `api/`.
   => `/api/auth/*` requests fail; both login and Google flows dead.
2. **Secrets absent at runtime** — `app.ts:1`/`server.ts:1` rely on `dotenv/config`, but `.env`
   is gitignored and not present on Vercel. Missing `MONGODB_URI` -> db falls back to
   `localhost` (`db.ts:153`) -> connection error -> 500 on login/register.
   Missing `GOOGLE_CLIENT_ID`/`SECRET` -> 400 from `/google/url` and thrown error in callback.
3. **OAuth redirect_uri mismatch** — `auth.ts:193-195` and `:235-237` build the redirect URI
   dynamically from `req.get("host")` (the live Vercel domain). Works only if that exact URI
   is registered in Google Cloud Console.
4. **Latent**: `api/index.ts` exports an Express instance, not a `(req,res)` handler Vercel
   expects; DB connection (`db.ts:431`) is fired async with no caching/promise reuse across
   serverless cold starts.

## Implementation steps
1. **Build the API into Vercel's `api/`**
   - Change build so a serverless handler exists as compiled JS, e.g. add an esbuild step:
     `esbuild api/index.ts --bundle --platform=node --format=cjs --packages=external --outfile=api/index.js`
     (or point Vercel to a `build` output). Confirm `api/index.js` is the entry.
   - Update `vercel.json` to declare the function:
     ```json
     "functions": { "api/index.js": { "runtime": "nodejs20.x", "includeFiles": "dist/**" } }
     ```
   - Ensure `api/index.js` exports a handler: `export default async (req, res) => app(req, res);`
2. **Set environment variables in Vercel dashboard** (Project > Settings > Environment Variables):
   `MONGODB_URI`, `JWT_SECRET`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`.
   Also set `GOOGLE_REDIRECT_URI` to the production callback to avoid host-based guesswork.
3. **Register production OAuth callback** in Google Cloud Console:
   `https://<your-vercel-domain>/api/auth/google/callback` (and keep `localhost` for dev).
4. **Cache the MongoDB connection** so it survives serverless cold starts (module-level
   promise, reuse if `mongoose.connection.readyState === 1`).

## Validation
- `vercel dev` locally: hit `/api/health` (expect `{status:"ok"}`).
- `/api/auth/register` with a test email -> 201 + token.
- `/api/auth/google/url` -> 200 with a URL whose `redirect_uri` matches the registered one;
  complete the popup flow -> success message + dashboard redirect.
- Check Vercel Function logs for DB connection / missing-env errors.
